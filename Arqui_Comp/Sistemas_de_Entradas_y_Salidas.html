Explora los mecanismos que permiten la comunicación entre el procesador y los dispositivos externos

¿Qué son los sistemas de I/O?

Los sistemas de Entrada/Salida (I/O) son los mecanismos que permiten la comunicación entre el procesador (CPU) y los dispositivos externos como teclados, pantallas, discos duros, redes y más.

Constituyen un componente esencial en la arquitectura de computadores. A través de mecanismos como interrupciones, acceso directo a memoria (DMA) y buses de comunicación, se garantiza una transferencia eficiente de datos y una adecuada gestión de los recursos del sistema.

Un sistema de I/O está formado por hardware (controladores, buses, puertos), firmware (drivers embebidos) y software (sistema operativo). Cada dispositivo tiene un controlador que actúa como intermediario entre el dispositivo y la CPU.

¿Por qué es importante?

Los sistemas de entrada/salida son fundamentales porque determinan la eficiencia con la que se comunican los componentes del computador, influyendo directamente en el rendimiento general.

Una CPU puede ser extremadamente rápida, pero si los sistemas de I/O son lentos, el cuello de botella impedirá que el usuario note los beneficios de ese procesador potente. Esto es especialmente crítico en entornos modernos como la inteligencia artificial y los centros de datos.

Evolución de los Sistemas de I/O

Década 1970-1980

Primeros microprocesadores. I/O básico mediante puertos serie y paralelo. Sin interrupciones sofisticadas.

Década 1990-2000

Introducción de USB, FireWire. DMA más común. Controladores integrados en chipset.

Década 2010-2026

PCIe 7.0, USB4, Thunderbolt, CXL. I/O de alta velocidad integrados directamente en la CPU.

Tecnologías 2026

En 2026, los sistemas de I/O se han optimizado mediante tecnologías de alta velocidad que permiten mayores tasas de transferencia, mejor eficiencia energética y un mejor soporte para aplicaciones exigentes como la IA y el cómputo de alto rendimiento.

Interrupciones (IRQ – Interrupt Request / Solicitud De Interrupción)

Las interrupciones (IRQ, Interrupt Request) son un mecanismo fundamental en los sistemas de entrada/salida que permite a los dispositivos periféricos notificar a la CPU, de forma asíncrona, cuando requieren atención. De esta manera, se evita que el procesador tenga que consultar constantemente el estado de los dispositivos como ocurre en técnicas como polling o busy-waiting, optimizando el uso de los recursos y mejorando la eficiencia del sistema.

Tipos de Interrupciones

Por Origen

  • Hardware: Dispositivos físicos (teclado, disco, red)
  • Software: Instrucciones INT n (syscalls)
  • Excepciones: Errores CPU (división por cero, page fault)

Por Activación

  • Edge-triggered: Flanco (0→1 o 1→0)
  • Level-triggered: Nivel (HIGH mientras dura)
  • * Legacy típicamente level
  • * PCIe moderno edge

Enmascarables

  • • IRQ estándar (0-15)
  • • Se pueden habilitar/deshabilitar
  • • Con flag IF en CPU
  • • CLI/STI en x86

No Enmascarables (NMI)

  • • Siempre atendidas
  • • Paridad de memoria
  • • Fallo de Hardware
  • • No se puede ignorar

¿Cómo funciona?

  • 1. El dispositivo periférico (teclado, mouse, disco duro, tarjeta de red, GPU, etc.) genera una señal eléctrica llamada IRQ cuando necesita atención.
  • 2. El controlador de interrupciones (APIC) recibe la señal, la prioriza y la envía a la CPU correspondiente.
  • 3. La CPU suspende la ejecución del programa actual, guarda el contexto (registros y contador de programa) en la pila.
  • 4. Ejecuta la ISR (Rutina de Servicio de Interrupción) correspondiente al dispositivo.
  • 5. Al terminar la ISR, restaura el contexto y continúa exactamente donde se había interrumpido.

Todo este proceso ocurre en cuestión de microsegundos, siendo prácticamente imperceptible para el usuario.

¿Qué es Polling?

El polling es un método de espera activa en el que la CPU consulta repetidamente el estado de un dispositivo para determinar si requiere atención.

Ejemplo

Imagina que estás esperando un mensaje:

  • Polling: Tú miras el celular cada 2 segundos, aunque no haya nada.
  • Interrupción: El celular te notifica solo cuando llega algo.

Problema

  • Gasta tiempo de la CPU.
  • Es ineficiente.
  • Consume más energía.

Por eso existen las interrupciones: el dispositivo avisa cuando pasa algo y la CPU no pierde tiempo.

Tarjetas de Red (NIC)

La NIC (Network Interface Card) es el componente que permite la comunicación con una red, ya sea por cable (Ethernet) o inalámbrica (WiFi).

Ethernet (Cableada)

  • • Datos via señales eléctricas por cable
  • • Más estable y menor latencia
  • • Menos interferencias

WiFi (Inalámbrica)

  • • Datos via ondas de radio
  • • Más flexible (sin cables)
  • • Mayor latencia, posibles interferencias

Cuando llegan paquetes, la NIC genera una IRQ para notificar a la CPU. Ethernet genera menos interrupciones (mejor coalescing), mientras WiFi genera más por el medio inalámbrico ruidoso.

Polling vs Interrupciones vs DMA

Los 3 métodos para manejar I/O. Cada uno tiene un costo diferente en términos de overhead de CPU:

Método Overhead CPU Latencia Uso típico
Polling Muy Alto Alta (consulta continua) Dispositivos simples, embedded
Interrupciones Bajo Baja (evento inmediato) Teclado, mouse, red
DMA Mínimo Muy baja (hardware) SSD, GPU, audio, red masiva

Nota: Polling consume CPU continuamente. Interrupciones solo cuando hay evento. DMA transfiere sin CPU (solo setup inicial).

Controladores de Interrupción

Los controladores de interrupción gestionan las solicitudes de IRQ de los dispositivos y las enrutan a la CPU.

PIC (8259A) - Legacy

  • • 8 líneas de IRQ (IRQ 0-7)
  • • 2 PICs en cascada (maestro/esclavo)
  • • Usado hasta Pentium
  • • Programable (ICW1-4)

APIC (Advanced PIC) - Moderno

  • • Hasta 24 líneas IRQ
  • • Local APIC en CPU
  • • IOAPIC para periféricos
  • • Soporte multi-núcleo

Funcionamiento del APIC

  1. Dispositivo genera IRQ → controlador IOAPIC
  2. IOAPIC crea mensaje APIC (vector 0-255)
  3. Se envía por APIC bus al Local APIC
  4. Local APIC interrumpe a la CPU
  5. CPU ejecuta ISR usando el vector

Acceso Directo a Memoria (DMA)

El DMA es una técnica que permite transferir datos entre un dispositivo y la memoria RAM sin que la CPU tenga que intervenir en cada operación.

¿Por qué es necesario?

Programmed I/O (Entrada/Salida Programada)

Método donde la CPU controla directamente la transferencia de datos.

Ejemplo
Transferir 1 MB:
≈ 15 millones de ciclos CPU
Problema
  • • La CPU ejecuta una instrucción por cada dato
  • • Se vuelve lenta e ineficiente
Resultado:
CPU prácticamente bloqueada
Solución con DMA

Con DMA, la CPU solo interviene en la configuración inicial y recibe una IRQ al final. Esto libera la CPU para otras tareas mientras ocurre la transferencia.

¿Cómo funciona DMA?

Solicitud DRQ: El dispositivo I/O solicita transferencia al controlador DMA, no a la CPU.

Controlador DMA: Un chip especializado que actúa como intermediario entre el dispositivo I/O y la memoria.

Transferencia Directa: Los datos van directamente de dispositivo a RAM sin pasar por la CPU.

Interrupción Final: Al terminar, el DMA envía una IRQ para notificar a la CPU.

Beneficios cuantitativos

• CPU involucrada solo en la configuración inicial y la IRQ final (aprox. 200 ciclos).
• En una transferencia de 1 MB, la CPU queda libre más del 99.9% del tiempo.
• Ideal para SSD NVMe, tarjetas de red, audio y GPUs.

Componentes clave del DMA

Controlador DMA: Gestiona direcciones de memoria, contadores y control de transferencia.

Ciclos de Bus: DMA roba ciclos del bus de sistema para transferir bloques de datos.

Demultiplexor de Señales: Coordina entre CPU, memoria y periféricos sin conflictos.

Modos de Transferencia: Burst (rápido), Cycle Stealing (intercalado) o Transparent (sin interferencia).

Modos de Transferencia DMA

Burst Mode

  • • Transfiere todo el bloque seguido
  • • CPU no puede usar el bus
  • • Más rápido
  • • Puede causar latencia

Cycle Stealing

  • • Roba ciclos al CPU
  • • interleaved con CPU
  • • Balance velocidad/CPU
  • • Más común

Transparent Mode

  • • Solo cuando CPU no usa bus
  • • Sin interferencia
  • • Más lento
  • • Eficiencia garantizada

Jerarquía de Buses

Los buses son los canales de comunicación que transportan datos entre los componentes del computador. En arquitecturas modernas, se organizan en una jerarquía según su velocidad y cercanía a la CPU.

¿Por qué existe una jerarquía?

Porque hay una gran diferencia entre la velocidad de la CPU y la de los dispositivos.

Entonces:

  • Componentes cercanos → más rápidos
  • Componentes lejanos → más lentos

PCIe Root Port vs Switch

Root Port (Puerto Raíz)

  • • Definición: Punto de conexión principal del Root Complex (CPU/Chipset) al bus PCIe.
  • • Función: Origina transacciones PCIe, es el "dueño" del bus.
  • • Relación: Un CPU tiene varios Root Ports (ej: x16, x4, x1).
  • • Analogía: Es como el puerto principal de un concentrador eléctrico.

PCIe Switch

  • • Definición: Dispositivo que conecta múltiples buses PCIe, permitiendo compartir enlaces.
  • • Función: Dirige tráfico entre dispositivos conectados, como un switch de red.
  • • Uso: Permite conectar varias GPUs o SSDs a un solo slot x16.
  • • Analogía: Es como un multiplexor que divide un camino en varios.

En resumen: El Root Port es el origen (donde nace el bus), y el Switch es el distribuidor que divide ese bus en múltiples conexiones.

Tipos de Buses

Buses del Sistema

  • FSB (Front Side Bus) - Legacy Intel
  • QPI (QuickPath Interconnect) - Intel
  • DMI (Direct Media Interface) - CPU↔PCH

Buses de Expansión

Buses Periféricos

Buses de Memoria

Ej: PCI, DDR antigua, FSB

Arbitraje del Bus

Cuando múltiples dispositivos quieren usar el bus al mismo tiempo, se necesita un mecanismo para decidir quién tiene acceso.

Daisy Chain

  • • Prioridad por posición física
  • • Dispositivo más cercano = mayor prioridad
  • • Simple pero no configurable

Priority Based

  • • priority configurable
  • • Dispositivos críticos primero
  • • Programable

Round Robin

  • • Turnos rotativos
  • • Fairness garantizado
  • • Sin inanición

PCIe: Credit-Based Flow Control

PCIe no usa arbitraje tradicional. Usa flow control credits: cada dispositivo tiene "créditos" para enviar paquetes. Se renuevan cuando el receptor confirma recepción. Esto evita perdido y elimina necesidad de arbitraje.

Avances Tecnológicos 2026

¿Qué es PCIe? (Peripheral Component Interconnect Express / Interconexión de Componentes Periféricos Express)

PCI Express (PCIe) es un bus de comunicación serie de alta velocidad que conecta componentes dentro del computadora. Utiliza un esquema de lanes (caminos de datos): un slot x1 tiene 1 lane, x4 tiene 4, x8 tiene 8 y x16 tiene 16. A más lanes, mayor ancho de banda. Es el estándar principal para GPUs, SSDs NVMe y tarjetas de expansión en 2026.

¿Qué es USB?

USB (Universal Serial Bus) es el estándar de conectividad más utilizado para conectar periféricos externos a una computadora. Fue creado para simplificar la conexión de dispositivos como teclados, mouse, impresoras, discos externos y cargadores.

Thunderbolt: Desarrollado inicialmente por Intel en colaboración con Apple (primera en implementarlo en Mac en 2011), combina PCIe y DisplayPort en un solo cable. Thunderbolt 5 alcanza 80 Gbps, el mismo rendimiento que USB4 v2.

Apple y Thunderbolt: Apple sigue siendo muy "celosa" con Thunderbolt. En los MacBook Pro con M5 Pro y M5 Max (2026) implementa Thunderbolt 5 con controladores propios integrados en el chip, ofreciendo una de las versiones más potentes y estables del mercado (soporte para más monitores externos, mejor ancho de banda por puerto). Los modelos base (como MacBook Air M5) siguen en Thunderbolt 4 / USB4 de 40 Gbps.

Importancia: En 2026, USB4 v2 alcanza 80 Gbps, rivalizando con PCIe en velocidad pero manteniendo la facilidad de uso hot-plug (conectar sin apagar el equipo).

¿Qué es CXL?

CXL (Compute Express Link) es una tecnología de interconexión de alta velocidad diseñada para centros de datos. Permite conectar CPUs, GPUs, aceleradores y memoria de manera coherente, compartiendo datos sin copias intermedias.

Importancia: En 2026, CXL 4.0 alcanza 128 GT/s, habilitando memoria compartida entre múltiples dispositivos para workloads de IA.

Origen: CXL fue desarrollado inicialmente por Intel, quien donó la tecnología al CXL Consortium en 2019 para convertirla en un estándar abierto.

Gobierno: Al igual que PCIe o USB, ninguna empresa es dueña exclusiva de CXL. Es gobernado por el Consortium (donde participan Intel, AMD, NVIDIA, Google, Microsoft, etc.), y cualquier compañía puede implementarlo de forma interoperable.

Integración de Inteligencia Artificial en dispositivos de entrada/salida para procesar datos localmente sin depender totalmente de la CPU.

El I/O inteligente permite que los dispositivos procesen datos usando IA, reduciendo la carga de la CPU. Por ejemplo, las SmartNICs optimizan el tráfico de red de forma autónoma.

¿Qué son el Ancho de Banda y la Latencia en I/O?

Ancho de Banda

Cantidad máxima de datos que pueden transferirse por segundo a través de un bus o canal de comunicación. Se mide en bits por segundo (bps) o múltiplos como GB/s (gigabytes por segundo).

Ejemplo: PCIe 7.0 x16 ofrece hasta 256 GB/s de ancho de banda bidireccional.

Latencia

Tiempo que transcurre desde que se solicita una transferencia de datos hasta que los primeros datos comienzan a llegar. Se mide en nanosegundos (ns) o microsegundos (µs).

Baja latencia = respuesta más rápida. Crítica para IA, bases de datos y juegos.

🧊 Analogía del agua

Imagina una manguera: el ancho de banda es el diámetro de la manguera (cuánta agua pasa a la vez), y la latencia es el tiempo que tarda el agua en llegar desde el otro extremo.

🚗 Analogía del tráfico

El ancho de banda es como el número de carriles de una autopista. Más carriles = más autos pueden pasar simultáneamente. Sin embargo, si hay un atasco (latencia), los autos tardan más en llegar a destino.

📊 Relación Ancho de Banda vs Latencia

  • • Alto ancho de banda + baja latencia = Ideal (PCIe 7.0 para IA)
  • • Alto ancho de banda + alta latencia = Aceptable para transferencias masivas (USB para discos)
  • • Bajo ancho de banda + baja latencia = Bueno para comandos (teclado, red)
  • • Bajo ancho de banda + alta latencia = Problema de rendimiento

⚡ GB/s vs GT/s: ¿Cuál es la diferencia?

GB/s (Gigabytes por segundo)

Mide la cantidad de datos útiles transferidos por segundo. Representa los datos reales que se mueven después de eliminar la sobrecarga (overhead) de codificación.

  • • 1 GB/s = 1,000,000,000 bytes/s
  • • Es la velocidad real de transferencia
  • • Lo que ves en la práctica
GT/s (Gigatransferencias por segundo)

Mide las transacciones físicas por segundo en el bus. Es la velocidad "bruta" antes de aplicar codificación y overhead.

  • • 1 GT/s = 1,000,000,000 transferencias/s
  • • Es la velocidad señal del bus
  • • No es lo que experimentas directamente
📌 Ejemplo práctico PCIe 7.0

256 GT/s (velocidad de señal) frente a ~242 GB/s (datos útiles) usando codificación 128b/130b.

La diferencia (~5%) corresponde al overhead de codificación. Por eso PCIe 7.0 x16 anuncia "256 GT/s", pero logra aproximadamente 512 GB/s bidireccionales al sumar ambos sentidos de transmisión.

PCIe

  • • Propósito: Conexión interna de componentes
  • • Ventaja: Mayor ancho de banda
  • • Usa: GPUs, SSDs NVMe, tarjetas
  • • Hot-plug: Solo con energía

CXL

  • • Propósito: Coherencia CPU-GPU-Memoria
  • • Ventaja: Memoria compartida
  • • Usa: Servidores, IA, HPC
  • • Latencia: ~100ns ultra baja

USB

  • • Propósito: Periféricos externos
  • • Ventaja: Hot-plug fácil
  • • Usa: Teclados, docks, cargas
  • • Distancia: Hasta 3-5m

Resumen: PCIe es para dentro del equipo (máxima velocidad), CXL es para servidores con coherencia de memoria, y USB es para conectar dispositivos externos fácil.

🧬 PCIe (Peripheral Component Interconnect Express)

  • Uso: General
  • Función: Conectar dispositivos (GPU, SSD, red, etc.)
  • Dónde se usa: PCs, laptops, servidores
  • Tipo de comunicación: Transferencia de datos
  • Memoria compartida: ❌ No (cada dispositivo usa su propia memoria)
  • Estado: Estándar ampliamente adoptado

📌 Es como una autopista rápida donde los dispositivos envían paquetes, pero cada uno vive en su propia "casa".

🧬 CXL (Compute Express Link)

  • Uso: Especializado
  • Función: Compartir memoria entre CPU, GPU y aceleradores
  • Dónde se usa: Principalmente servidores y centros de datos
  • Tipo de comunicación: Datos + memoria coherente
  • Memoria compartida: ✅ Sí (memoria unificada)
  • Estado: Tecnología emergente y avanzada

📌 Aquí ya no hay casas separadas: todos comparten la misma "memoria", como si fuera una mente colectiva.

La Tarjeta Madre y su Relación con PCIe

La tarjeta madre (motherboard) es el circuito impreso principal que conecta todos los componentes de la computadora. En ella se encuentran los slots PCIe donde se instalan tarjetas de expansión como GPUs, SSDs NVMe y tarjetas de red.

Slots PCIe en la Motherboard

  • • Slot x16: Para GPUs de alto rendimiento. Proveen el máximo ancho de banda (hasta 512 GB/s en PCIe 7.0 x16).
  • • Slot x4: Comúnmente usado para SSDs NVMe Gen4/Gen5. También para tarjetas de expansión.
  • • Slot x1: Para tarjetas de sonido, red Gigabit, controladoras USB adicionales.

Puertos Integrados

  • • USB: Puertos USB-A y USB-C integrados en el chipset (PCH). USB4 v2 hasta 80 Gbps.
  • • SATA: Para discos duros y SSDs SATA. Legacy pero aún presente (6 Gbps).
  • • Ethernet: Controladora de red integrada (2.5 GbE estándar en 2026).
  • • HDMI/DisplayPort: Video integrado (desde la GPU integrada o CPU).

En resumen: La tarjeta madre es el "hub" central que conecta CPU, RAM, almacenamiento y periféricos. Los slots PCIe son las autopistas de alta velocidad que permiten comunicación directa entre la CPU y dispositivos de alto rendimiento como GPUs y SSDs.
